# The Generalized Pirate Game (推广版海盗分金币问题)
原题引用：<https://en.wikipedia.org/wiki/Pirate_game>

本目录包含了广义海盗分金币问题（N 个海盗，M 枚金币）的逆向归纳数学证明以及 Python 算法博弈论模拟。

## 题目设定

有 $N$ 个绝对理性、极度聪明的海盗，按照资历从老到新编号为 $N, N-1, \dots, 2, 1$（$N$ 号资历最老，$1$ 号资历最年轻）。  
他们需要分配 $M$ 枚金币（经典版本中 $M=100$）。

分配规则：
1. 由当前资历最老的海盗提出一个金币分配方案。
2. 所有存活的海盗（包括提案者本人）对该方案进行表决。
3. 如果赞成票数≥总存活人数的一半，方案通过，按方案分配金币，游戏结束。
4. 如果赞成票数<一半，提案者会被扔进海里喂鲨鱼。由剩下海盗中资历最老的接替，从步骤一重新开始。

海盗的效用函数（优先级排序）为：
**保命** > **获得更多金币** > **嗜血（收益相同时优先把别人扔下海）**

当海盗数量 $N$ 远大于金币数量 $M$ 时，会发生什么？

## 最优策略的三个相变阶段

当 $M=100$ 时，随着海盗数量 $N$ 的增加，系统会经历三次相变：

- **第一阶段（N ≤ 200）**：老大永远存活，通过精准收买刚好能拿到半数选票，剩余金币全归自己。
- **第二阶段（N = 201 或 202）**：老大为了保命必须把所有金币分发出去，自己没拿到金币，但能存活。
- **第三阶段（N > 202）**：金币已经不足以买到足够的选票。海盗的存活完全依赖于后续海盗对死亡的恐惧导致的免费赞成票。在此阶段，只有特定编号的海盗能够存活，其余海盗无论怎么做都必死无疑。

## 数学证明 (逆向归纳法)

设当前存活的海盗总数为 $N$，金币总数为 $M$。提案者 $N$ 号需要获得的最低赞成票数为：

```math
V_{required} = \lceil \frac{N}{2} \rceil
```

### 阶段一：$N \le 2M$

当 $N \le 2M$ 时，提案者 $N$ 自己投自己 1 票，他还需要购买的选票数为：

```math
V_{buy} = \lceil \frac{N}{2} \rceil - 1
```

由于海盗具有嗜血属性，要收买一个海盗，必须给他比在 $N-1$ 轮中预期收益多至少 1 枚金币。
根据逆向归纳，提案者 $N$ 只需要找出在 $N-1$ 轮中收益为 $0$ 的海盗，给他们每人 $1$ 枚金币即可获得赞成票。
因此，提案者 $N$ 最终自己持有的金币数为：

```math
Profit_{N} = M - \left( \lceil \frac{N}{2} \rceil - 1 \right)
```

### 阶段二：$N = 2M + 1$ 或 $2M + 2$

以 $M=100$ 为例。
当 $N = 201$ 时，需要 $101$ 票。自己 $1$ 票，剩下 $100$ 枚金币全部分给在 $200$ 号方案中拿不到金币的 $100$ 个人。获得 $101$ 票，存活，收益为 $0$。
当 $N = 202$ 时，需要 $101$ 票。自己 $1$ 票，把 $100$ 枚金币分给在 $201$ 号方案中拿不到金币的 $100$ 个人。获得 $101$ 票，存活，收益为 $0$。

### 阶段三：$N > 2M + 2$

当 $N \ge 203$ 时，金币已经不够买选票了。存活的唯一希望是那些如果这个提案被否决后在下一轮必死的海盗，为了活命会投出免费的赞成票。

- 对于 203 号：需要 102 票。自己 1 票，100 枚金币买 100 票，共 101 票。202 号有存活方法因此没有人会送他免费票，203 号必死。
- 对于 204 号：需要 102 票。自己 1 票，100 枚金币买 100 票。因为 203 号知道如果 204 号死了，接下来当老大的自己必死，因此 203 号为了活命会免费投给 204 号 1 票。总票数：

```math
1 (\text{self}) + 100 (\text{bought}) + 1 (\text{from 203}) = 102
```

因此 204 号存活。

顺着此逻辑推演：
- 205 号：需要 103 票。自己(1) + 购买(100) + 免费票(0，因为如果 205 死，204 会存活，204 不存在必死威胁)。仅 101 票，必死。
- 206 号：需要 103 票。自己(1) + 购买(100) + 免费票(1，来自必死的 205)。仅 102 票，必死。
- 207 号：需要 104 票。自己(1) + 购买(100) + 免费票(2，来自必死的 206, 205)。仅 103 票，必死。
- 208 号：需要 104 票。自己(1) + 购买(100) + 免费票(3，来自必死的 207, 206, 205)。总票数：

```math
1 (\text{self}) + 100 (\text{bought}) + 3 (\text{free}) = 104
```

因此 208 号存活。

### 规律

经过递推，我们发现所有存活的海盗所需的免费选票数量正好跨越了一个 $2$ 的幂次方区间。
设能够存活的海盗编号构成的集合为 $S$，则有：

```math
S = \{ N \mid N = 2M + 2^k, \quad k \in \mathbb{N} \} \cup \{ 2M + 1 \}
```

因此，在 $100$ 枚金币的条件下，大于 $200$ 号之后能存活的海盗编号必定满足：

```math
N = 200 + 2^k
```

即存活序列严格收敛于：201, 202, 204, 208, 216, 232, 264, 328...
处于序列间隔中的所有海盗，无论金币如何分配，都将面临被扔入海中喂鲨鱼的命运。

Q.E.D.
