# 100 Prisoners Problem

原题引用：<https://en.wikipedia.org/wiki/100_prisoners_problem>

本目录包含了 100 prisoners problem 的数学证明以及 Python 模拟验证。

## 题目

有 100 个囚犯，编号 1–100。  
有 100 个盒子，里面随机放着 1–100 的编号纸条，每个盒子一张。  
每个囚犯可以单独进入房间，最多打开 50 个盒子，之后盒子恢复原状，不能和后面的人交流。  
如果所有囚犯都找到自己的编号，全部获释；只要有一个没找到，全部处死。

囚犯们的最优策略是什么？如何证明？

## 最优策略：cycle-following strategy

对于任意囚犯 i：

- 打开编号为 i 的盒子；如果得到编号为 i 的纸条，则成功。
- 如果打开编号为 i 的盒子得到编号为 j 的纸条（j ≠ i），则去打开编号为 j 的盒子。
- 如此重复，最多进行 50 次。

结果是，这个最优策略有惊人的 31.183% 的概率使所有囚犯都能获释。

## 证明

下面我们证明为什么这个最佳策略如此有效。

如果了解一些组合数学或者抽象代数的知识，就会知道循环表示法（轮换表示法）。在这个证明中，将使用这种简洁的表示法。

设 f(x) 表示编号为 x 的盒子里装着的纸条编号。  
因为编号 1–100 都恰好只出现一次，所以 f 是集合

```math
X = \{1, \dots, 100\}
```

到其自身的双射，即 f : X → X。也就是说，f 是集合 X 上的一个置换。

这个策略的过程可以抽象为：对于囚犯 i，

```math
i, f(i), f(f(i)), \dots
```

囚犯 i 成功，当且仅当包含 i 的循环长度满足 ≤ 50。

对于任意 i ∈ X，所有囚犯都成功，当且仅当置换 f 的所有循环长度都满足 ≤ 50。

因此，

```math
P(\text{所有循环长度} \le 50)
=
1 - P(\text{存在某个循环长度} > 50)
```

这两类事件是互斥的，因为不可能同时存在两个长度都大于 50 的循环，否则集合元素总数会超过 100。

所以我们只需要计算失败的概率，再用 1 减去它，就能得到成功概率。

对于某个长度 k > 50 的循环，有

```math
\binom{100}{k}
```

种方法选出这 k 个元素。

如果不考虑循环等价，会有 k! 种排列；但由于循环在旋转意义下等价，所以需要除以 k，因此共有

```math
(k - 1)!
```

种不同的循环。

剩下的 (100 - k) 个元素可以任意排列，形成一个或多个长度 ≤ 50 的循环，共有

```math
(100 - k)!
```

种方法。

因此，包含一个长度为 k 的循环的置换总数为

```math
\binom{100}{k} \cdot (k - 1)! \cdot (100 - k)!
```

再除以所有置换总数

```math
100!
```

得到该事件的概率为

```math
\frac{1}{k}
```

于是，

```math
P(\text{失败}) = \sum_{k=51}^{100} \frac{1}{k}
```

所以，

```math
P(\text{成功}) = 1 - \sum_{k=51}^{100} \frac{1}{k} \approx 0.31183
```

Q.E.D.
